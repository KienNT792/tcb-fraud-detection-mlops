# TCB Fraud Detection MLOps - Tóm tắt project

## Tổng quan

Project này xây dựng một hệ thống phát hiện gian lận giao dịch thẻ theo hướng MLOps, với trọng tâm là:

- Pipeline xử lý dữ liệu và huấn luyện model `XGBoost`
- Đánh giá model theo các metric phù hợp cho dữ liệu mất cân bằng như `PR-AUC`, `F1`, `Recall`
- Serving realtime qua `FastAPI`
- Chuẩn bị sẵn chỗ cho orchestration, monitoring và CI/CD

Về mặt kiến trúc, repo được chia thành 3 lớp chính:

1. `ml_pipeline/`: xử lý dữ liệu, train, evaluate, inference logic
2. `serving_api/`: API phục vụ dự đoán online
3. `dags/`, `monitoring/`, `.github/workflows/`: phần vận hành, hiện mới ở mức scaffold

## Luồng xử lý chính

Luồng offline dự kiến của hệ thống:

1. `preprocess.py`
2. `train.py`
3. `evaluate.py`
4. Xuất artifact sang thư mục `models/`

Luồng online:

1. FastAPI khởi động
2. `serving_api/app/model_loader.py` load `FraudDetector`
3. `FraudDetector` dùng lại logic feature engineering từ pipeline training
4. API trả về `fraud_score`, `is_fraud_pred`, `threshold`, `risk_level`

## Thành phần chính theo thư mục

### `ml_pipeline/src/`

- `preprocess.py`:
  - Đọc raw CSV
  - Validate schema
  - Xử lý missing values và duplicate
  - Encode một số trường categorical / boolean dạng nghiệp vụ
  - Chia train/test theo thời gian để tránh leakage
  - Fit feature generators trên train only
  - Sinh feature như `transaction_hour`, `amount_log`, `segment_encoded`, `customer_tx_count`, `customer_avg_amount`
  - Lưu `train.parquet`, `test.parquet`, `features.json` và feature state

- `train.py`:
  - Đọc processed artifacts
  - Tính `scale_pos_weight` cho class imbalance
  - Train `XGBClassifier`
  - Tìm threshold tối ưu với ràng buộc ưu tiên `recall >= 0.95`
  - Lưu model, metrics, feature importance
  - Log run lên `MLflow`

- `evaluate.py`:
  - Đánh giá lại model trên test set
  - Phân tích threshold
  - So sánh với baseline
  - Sinh `segment_report.csv`
  - Sinh biểu đồ `PR curve` và `SHAP`
  - Ghi báo cáo vào `models/evaluation/evaluation.json`

- `inference.py`:
  - Cài đặt class `FraudDetector`
  - Load toàn bộ artifact một lần
  - Reuse đúng feature engineering như lúc train
  - Hỗ trợ predict một giao dịch hoặc batch
  - Trả thêm `risk_level` theo 3 mức: `LOW`, `MEDIUM`, `HIGH`

### `serving_api/`

- `app/main.py` cung cấp các endpoint:
  - `GET /`
  - `GET /health`
  - `POST /predict`
  - `POST /predict/batch`

- `app/schemas.py` định nghĩa request/response schema bằng `Pydantic`
- `app/model_loader.py` quản lý singleton `FraudDetector` để không load model lại ở mỗi request

## Hiện trạng thực tế của repo

Project đã có phần lõi ML và API tương đối rõ:

- Logic preprocessing, training, evaluation, inference đã có code
- Có artifact model và report đánh giá được commit sẵn
- Có test cho preprocessing và API

Tuy nhiên vẫn còn các phần chưa hoàn thiện hoặc mới là placeholder:

- `dags/fraud_pipeline.py` đang trống
- `monitoring/evidently_ai/drift_monitor.py` đang trống
- `.github/workflows/ci-cd-pipeline.yml` đang trống
- `docker-compose.yml`, `Makefile`, `docs/api_docs.md`, `README.md` đang trống

Ngoài ra, trong workspace hiện tại không có thư mục `data/` được commit kèm, nên muốn chạy end-to-end từ raw data sẽ cần bổ sung dữ liệu đầu vào và các artifact processed tương ứng.

## Kết luận ngắn

Đây là một repo MLOps cho bài toán fraud detection với phần model pipeline và serving API đã được dựng khá rõ, nhưng phần orchestration, monitoring, CI/CD và tài liệu vận hành vẫn chưa hoàn thiện. Nếu tiếp tục phát triển, ưu tiên hợp lý sẽ là:

1. Hoàn thiện Airflow DAG và CI/CD
2. Bổ sung monitoring drift / quality
3. Chuẩn hóa tài liệu chạy local và deploy
4. Thêm dữ liệu mẫu hoặc cơ chế bootstrap `data/processed`
